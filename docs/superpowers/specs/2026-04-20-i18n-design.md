# i18n Design: Add English Language Support

## Summary

Add internationalization (i18n) to the AI-novel project — frontend UI, backend API responses, and documentation. Chinese remains the default language; English is the added language.

## Decisions

- **Frontend library**: i18next + react-i18next + i18next-browser-languagedetector
- **Language switch**: Manual toggle in Sidebar (中/EN), persisted to localStorage
- **Default language**: zh, fallback: zh
- **Backend approach**: Simple dict-based translation with Accept-Language header detection
- **Documentation**: Dual files — README.md (English) + README_zh.md (Chinese), same for CONTRIBUTING

## Frontend Architecture

### Translation File Structure

```
frontend/src/locales/
├── zh/
│   ├── common.json        # Navigation, buttons, status labels, time formatting
│   ├── dashboard.json     # Dashboard page
│   ├── novels.json        # Novel management page
│   ├── novelDetail.json   # Novel detail page (largest, ~563 occurrences)
│   ├── chapterEditor.json # Chapter editor page
│   ├── config.json        # Config page
│   └── providers.json     # Provider config page
├── en/
│   └── (mirror structure)
```

### i18next Configuration

- Init in `frontend/src/i18n.ts`, imported in `main.tsx`
- Namespaces mapped 1:1 to pages; `common` loaded globally
- Browser language detection on first visit, localStorage persistence after
- No lazy-loading needed (only 2 languages, small payload)

### Key Translation Patterns

| Pattern | Chinese Example | Key Format |
|---------|----------------|------------|
| Static labels | "导入小说" | `"importNovel"` |
| Nested status | "进行中" | `"status.processing"` |
| Interpolation | "共 5 章" | `"chapterCount": "共 {{count}} 章"` |
| Relative time | "3 分钟前" | `"timeAgo.minutes": "{{count}} 分钟前"` |
| Scene types | "战斗", "对话" | `"sceneType.combat"`, `"sceneType.dialogue"` |

### Language Switcher Component

- Placed at bottom of Sidebar, above any existing footer content
- Simple toggle button: shows "EN" when current is zh, shows "中" when current is en
- Calls `i18n.changeLanguage()` on click

## Backend Architecture

### Translation Approach

- `backend/i18n/` directory with `zh.py` and `en.py` containing message dicts
- Utility function `t(key, lang)` for translation lookup
- Language detected from `Accept-Language` header via FastAPI middleware
- Scope: error messages, status labels, and any user-facing strings in API responses
- Internal logs remain in English (no translation needed)

### Middleware

- FastAPI middleware extracts language preference from `Accept-Language` header
- Stores in request state, accessible by route handlers
- Default: zh
